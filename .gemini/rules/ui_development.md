# UI Development & Stitch Design Rules

When interacting with UI designs, specifically those generated by the AI tool "Stitch" or provided via HTML blocks, you must strictly adhere to the following rules:

1. **Persistent TopBar:** The `TopAppBar.jsx` component is locked. Do not override, remove, or drastically alter the `TopAppBar` when implementing new page designs. 
2. **Scoping New Designs:** New HTML designs should only replace the `<main>` content boundary of the view component. Navigation remains global and consistent across the whole application located in `App.jsx`.
3. **Tailwind Framework:** The application natively runs Tailwind CSS v3. All provided HTML classes can be used directly without translation to Vanilla CSS.
4. **Listing and Detail Page Navigation Standard:**
   To ensure consistency, all main entity modules must follow this strictly standardized header button anatomy:
   - **Listing Pages Header (Right-Aligned Order):**
     1. **Delete All:** A square red outline icon button (`<button className="w-8 h-8 rounded border border-error/40 text-error flex items-center justify-center hover:bg-error/10 transition-colors"><span className="material-symbols-outlined text-[18px]">delete</span></button>`). Replace standard label buttons with this icon container.
     2. **Import/Secondary Action:** Standard secondary text button (e.g. `<Button variant="secondary">`).
     3. **Create/New Action:** Standard primary text button (e.g. `<Button variant="primary">`).
   - **Detail Pages Header (Right-Aligned Order):**
     1. **Dynamic Status Action:**
        - When `!isEditing`: A square Back button with outline styling, containing only a `chevron_left` icon and no text (`<Link className="w-8 h-8 rounded border border-primary/40 text-primary flex items-center justify-center hover:bg-primary/10 transition-colors" title="Back"><span className="material-symbols-outlined text-[18px]">chevron_left</span></Link>`).
        - When `isEditing`: A Delete entity button, using the exact square red outline icon format defined for the Listing pages above.
     2. **Edit/Save Toggle:**
        - When `!isEditing`: A primary color Outline button (`className="px-5 py-2 font-bold font-label text-xs uppercase tracking-widest transition-colors border border-primary/50 text-primary hover:bg-primary/10"`). Text = "Edit [Entity]".
        - When `isEditing`: A Solid Filled button (`className="px-5 py-2 font-bold font-label text-xs uppercase tracking-widest transition-colors bg-primary text-on-primary glow-primary"`). Text = "Save [Entity]".
